# 2025 年 LLM 的六个范式变化

2025 年对大语言模型而言，是进展强劲且事件密集的一年。

这篇文章总结了我认为最值得注意、也略带意外的"范式变化"。它们改变了行业版图，也在概念层面格外突出。

## 一、可验证奖励的强化学习（RLVR）

2025 年初，各家实验室的训练配方大致是这样的：

预训练 → 监督微调 → 人类反馈强化学习

这套配方稳定且有效。到了 2025 年，一个新的阶段被加入进来：**可验证奖励的强化学习（RLVR）**。

原理很简单：让 LLM 在"可自动验证"的环境中训练，比如数学谜题、代码挑战。结果很神奇——LLM 自发发展出了我们教不会它的"推理"能力：拆解问题、中间步骤、尝试纠错。

以前我们很难得到这些能力，因为我们根本不知道"最优的推理轨迹"应该长什么样。现在让模型自己去找，它反而找到了。

RLVR 带来的直接结果，是强化学习的训练时间显著拉长。OpenAI 的 o1 是首次展示，o3 则是明显拐点——你能直观感受到差异。

更重要的是，这带来了一个新的调节旋钮：通过增加"思考时间"，我们可以把能力与测试时计算量直接关联。让模型多想一会儿，它就会变聪明。

## 二、幽灵与动物：锯齿状智能

2025 年，我第一次真正"内化"了一个事实：**我们不是在培养动物，而是在召唤幽灵。**

人类的神经网络是为"丛林里的生存"而优化。LLM 的神经网络是为"模仿人类文本""在数学谜题里收集奖励"而优化。两者本质上不同。

结果是 LLM 呈现出令人发笑的"锯齿状"表现：它可以是博学的天才多面手，也可以是困惑的小学生；距离被一次越狱诱骗、把你的数据外泄，可能只差几秒钟。

这也让我对基准测试失去了信任。基准几乎在定义上就是"可验证环境"，因此会立刻受到 RLVR 的影响。在测试集上训练，正在成为一种新的艺术形式。

把所有基准都碾压了，但仍然得不到 AGI——这就是 2025 年的现状。

## 三、Cursor：新一层 LLM 应用

Cursor 的爆发，揭示了一个事实：**LLM 应用有一层新形态。**

人们开始讨论"X 领域的 Cursor"。这类应用把 LLM 调用打包并进行编排：负责上下文工程、在后台把多次调用串成复杂的 DAG、为人机交互提供特定 GUI、提供"自主性滑杆"。

关于这层应用到底有多"厚"，争论很多。我的看法是：LLM 实验室会培养出"通用能力较强的大学生"；而 LLM 应用则把这些"大学生"组织起来，通过注入私有数据、传感器、执行器和反馈回路，把它们真正"动画化"为能在具体领域落地的专业团队。

## 四、Claude Code：住在你电脑里的 AI

Claude Code 是我今年看到的第一个令人信服的 LLM 智能体。

它以带循环的方式，把工具使用与推理串联起来，进行长程问题求解。更重要的是，它运行在你的电脑上，能够直接使用你私有的环境、数据与上下文。

OpenAI 早期走了弯路：他们偏向云端容器部署、从 ChatGPT 编排。但在我们这个能力锯齿且起飞速度足够慢的中间阶段，把智能体直接运行在开发者电脑上更合理。

真正关键的差异，不在于 AI 计算跑在云端还是本地，而在于那台已经存在的电脑、它的环境、上下文、数据、密钥、配置，以及低延迟的人机交互。

Claude Code 改变了 AI 应用应该是什么样：它不再只是你访问的网站，而是一个"住在你电脑里"的小精灵。这是一种全新的人机交互范式。

## 五、氛围编程（Vibe coding）

2025 年，AI 跨过了一个能力阈值：**仅凭英文就能构建各种令人印象深刻的程序。**

我提出了 "vibe coding" 这个词，完全没意识到它会传播到什么程度。

在氛围编程下，编程不再严格属于受过高度训练的专业人士——任何人都可以做。这印证了我之前的观点：与此前几乎所有技术都相反，普通人从 LLM 中获得的收益，远大于专业人士。

氛围编程不仅赋能普通人"接近"编程，也会赋能专业人士写出大量原本根本不会被写出来的软件。我用氛围编程写过一个高效的 Rust BPE 分词器，而不必把 Rust 学到那个层级。

代码突然变得"免费"、短暂、可塑、用完即弃。氛围编程将重塑软件地貌。

## 六、Nano banana：LLM GUI 的雏形

Google Gemini Nano banana 是 2025 年最具范式冲击力的模型之一。

在我看来，LLM 是下一个重要的计算范式，类似于 1970、80 年代的计算机革命。因此我们会看到一系列相似类型的创新：个人计算的对应物、微控制器的对应物、互联网的对应物。

尤其在 UI/UX 层面，与 LLM "聊天"有点像 1980 年代在终端敲命令：文本是计算机偏好的原始数据表示，但并不是人类偏好的信息形态。

LLM 也应该用我们偏好的格式与我们交流——图像、信息图、幻灯片、白板、动画、Web 应用等。早期的雏形是 emoji 与 Markdown，但真正的 LLM GUI 会是什么样子？

nano banana 是一个早期信号。它不仅仅关乎图像生成，而是来自文本生成、图像生成与世界知识共同构成的联合能力。

## 总结

2025 年是一个令人兴奋的 LLM 之年。

LLM 正在成为一种新型智能：它们同时比我预期的要聪明得多，也比我预期的要愚蠢得多。无论如何，它们极其有用。

我认为整个行业甚至还没有实现其潜力的 10%。从概念上看，这个领域仍然是一片广阔的开放空间。

同时相信两件事：我们会继续看到快速而持续的进步，并且仍然有大量工作需要完成。

系好安全带。
